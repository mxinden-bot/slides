A Browser’s Perspective

Surprises and insights from Firefox networking telemetry.

MAPRG · IETF 126, Vienna · 23 July 2026  ·  Oskar Mansfeld & Max Inden · Mozilla

A client-side vantage point, at scale

  • Firefox reports anonymized networking telemetry from ~200M monthly active users (Firefox Public Data Report).
  • The client side, across the globe: it complements server-side and active-probing datasets.
  • Congestion control, DNS timings, path characteristics, QUIC adoption, Happy Eyeballs v3, and more.
  • Collected and aggregated with Glean, and entirely public: anyone can query it.

We use it every day to reason about the stack. Here are a few surprises, and how you can use it too.

How connections find HTTP/3

  • Only ~42% of connections are offered HTTP/3 at all.
  • Of those, ~84% learn it only from Alt-Svc, so after a first non-h3 connection: a wasted round trip.
  • An HTTPS record would move that discovery into DNS, before connecting, but almost nobody publishes one.

What HTTPS records actually carry

Where records exist, most advertise h3 and about half carry address hints, but only ~16% carry ECH, the feature that most needs them.

  • New connections start slow, and many carry too little data to ever leave slow start.
  • So high-rate send performance (the QUIC-upload debate) touches a small slice of traffic: worth measuring before optimizing.

ECN rarely survives the path

Only ~37% of connections see working ECN end to end: nearly half have marks bleached, another 16% are black-holed. A real constraint for L4S deployment.

Cutting through the noise

  • Nightly vs release. Nightly catches regressions within hours; release gives representative scale.
  • Roll out behind experiments. Compare arms on the same build, so the effect is the change, not the population.
  • Watch the denominator. Connections, requests, and page loads give different answers to “how much HTTP/3?”
  • The trap. A metric gated on an OS feature is buried by the majority OS. UDP GSO/GRO offload looks absent at os:"*" (Windows dominates), and only appears once you filter os:"Linux".

Takeaways

  • The client side, at global scale, is public: use it.
  • Surprises worth chasing: h3 discovery still wastes a round trip, ECH is barely deployed, most connections never leave slow start, ECN is mostly bleached.
