HTTPS has won

Encrypted Client Hello (ECH)

  • By default the TLS handshake sends the Server Name Indication (SNI) in clear text, e.g. wikipedia.org.
  • A major attack vector for censors today.
  • ECH encrypts parts of the TLS Client Hello, including the SNI.
  • Public key fetched via the DNS HTTPS record. Especially relevant behind CDNs.
  • Adoption still low. On TCP, Firefox attempts ECH ~0.3% of the time, and it nearly always succeeds.

QUIC and HTTP/3

  • General-purpose transport on top of UDP
  • Encrypted (meta)data
  • 1-RTT connection setup, 0-RTT on resumption
  • Reliable and unreliable delivery, no head-of-line blocking
  • Easy to evolve, often in user space

WebTransport: the next WebSocket?

  • A bidirectional, multiplexed transport for the Web, over HTTP/3.
  • Streams and unreliable datagrams.
  • No major consumer yet beyond upcoming Media over QUIC.

MASQUE: proxying on top of HTTP

  • Proxy protocol on top of HTTP; leverages QUIC unreliable datagrams.
  • Inherits HTTP’s censorship resistance.
  • Proxy UDP, TCP, IP, … (even Ethernet).
  • The extreme case of HTTP as the thin waist.

Oblivious HTTP

  • Similar to MASQUE, but request-oriented, not connection-oriented.
  • The relay sees the client IP, but not the request.
  • The destination sees the request, but not the client IP.
  • Used for Telemetry and Oblivious DoH.

Happy Eyeballs v3

A cardinality explosion of protocol combinations to choose from:

  • IPv4 / IPv6
  • TCP + TLS or QUIC
  • DNS A, AAAA or HTTPS records
  • HTTP alt-svc header
  • WebSocket or WebTransport

A set of heuristics to reduce connection-establishment latency while racing candidates.

Congestion control: ECN and L4S

  • Firefox’s QUIC stack does Cubic today; no BBR planned near-term.
  • ECN marking and echoing shipped: ~50% of network paths are ECN-capable.
  • Ready for an L4S congestion controller on the server side. (Hello, Comcast users.)
  • An ongoing research area.

What do we not see happening?

  • Old protocols (IPv4, DNS53, HTTP/1 & HTTP/2) going away.
  • An anonymous colleague: “Hopefully no HTTP/4.”

What can you do for a healthy Internet?

  • If you use a browser: enable DoH.
  • If you operate DNS infra: expose a DoH endpoint.
  • If you operate a website: serve it over HTTP/3 and add ECH.
