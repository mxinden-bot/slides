How connections learn about h3

  • 22% of connections learn h3 only via Alt-Svc, a round trip they need not pay.
  • Only 4.7% via an HTTPS record alone; 8% via both.
  • Motivation for publishing HTTPS records with alpn=h3.

Changes to the library

  • mozilla/happy-eyeballs: a deterministic, sans-I/O state machine, reusable outside Firefox.
  • Interleave protocol variant and address family together.
  • A configurable connection-attempt-delay multiplier (exponential back-off): Firefox runs 50 ms base ×2, not the 250 ms default.
  • A flag to skip waiting for the preferred family.

For the working group

  • The 250 ms Connection Attempt Delay is coarse; a smaller base with back-off works better in practice.
  • Would a shared, deterministic test suite for the algorithm help?
  • Deployment surfaces spec-adjacent questions: shuffling equal-priority SVCB records, and how racing interacts with connection coalescing across resolved addresses.
  • Open: proxy awareness (MASQUE connect-udp vs HTTP CONNECT ordering), WebSocket / WebTransport EXTENDED CONNECT.

Next steps

  • Ship Happy Eyeballs v3 to Firefox Release in the coming months.
  • Tune resolution and connection-attempt delays from the ~99th-percentile data.
  • More telemetry to share at IETF 126.
