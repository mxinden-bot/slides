Evolving HTTP/3 & QUIC

Growing usage beyond 30% of the Internet?

HTTP Workshop 2026 · discussion  ·  Max Inden · Mozilla

Is HTTP/3 really “beyond 30%”?

The neutral number has plateaued

  • Cloudflare’s likely-human request share sits ~30%, roughly flat over the past year (Cloudflare Radar).
  • Firefox’s own view is lower: HTTP/3 is ~17% of desktop requests, ~25% on Android (performance.mozilla.org).
  • It runs higher by other measures: site support ~40% (W3Techs), hyperscaler first-party far higher (Chrome↔Google >50%, Meta ~75%).
  • Requests vs connections is a real confound: HTTP/3 multiplexes many requests per connection.

QUIC v2: built to be deployed, barely deployed

  • RFC 9369 is functionally identical to v1: only crypto salts, labels and codepoints differ (anti-ossification).
  • Measured deployment: < 0.003% of QUICv1 domains; < 0.013% grease the QUIC bit (IMC 2024).
  • Firefox’s own telemetry agrees: of version-labeled QUIC handshakes in Nightly, v2 is a rounding error, v1 carries essentially all of them (GLAM: http_3_quic_version).
  • No feature payoff, so no individual incentive: adoption across open-source stacks is uneven.
  • Is greasing (RFC 9287) enough that v2 is unnecessary?

Alt-Svc → HTTPS RR: the upgrade path

  • Alt-Svc is learned after a slower HTTP/2 connection: the first visit never uses HTTP/3.
  • 29% of Firefox connections learn h3 only via Alt-Svc, a wasted round trip.
  • HTTPS records (RFC 9460) are ~15% adopted; ~70% one CDN’s defaults (IMC 2024).
  • ECH depends on the HTTPS record: now also a privacy feature, not just performance.

Do we deprecate Alt-Svc, or bridge it (draft-thomson-httpbis-alt-svcb)?

Optimistic QUIC, Happy-Eyeballs style

  • Race QUIC speculatively on the first connection, before any h3 signal.
  • curl 8.16 (Aug 2025): with HTTP/3 allowed, races QUIC then TCP after 200 ms. Still an opt-in, not automatic.
  • Risk: where UDP is blocked, an optimistic attempt is strictly slower (timeout, then TCP + TLS). Chrome’s QUIC failure rate runs ~6–12% by region (APNIC, 2025).
  • The right racing delay is unsettled and network-dependent.

Open questions

  • Which denominator should we agree on, and is the plateau the real worry?
  • Can QUIC v2 ever get deployed? Does a major library shipping no v2 matter?
  • Deprecate Alt-Svc for HTTPS RR, or bridge the two? Is ECH the reason sites finally publish HTTPS records?
  • What should the default be for optimistic QUIC, and who pays the fallback cost on UDP-hostile networks?
  • Is QUIC the best ground to deploy L4S? ECN is reflected natively in the ACKs, and the sender’s response lives in user space.
